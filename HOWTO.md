# 📘 GitHub 新手操作手册（平板实战版）

> 本文档适用于在平板（iPad/安卓）上，通过网页浏览器和 `vscode.dev` 操作 GitHub。

---

## 1. 网络问题怎么解决？
- **首选**：开手机热点给平板用（最稳定）。
- **备选**：WiFi + 修改 DNS 为 `1.1.1.1` 或 `8.8.8.8`。
- **现象**：如果网页打不开或报错 `ERR_CONNECTION_TIMED_OUT`，立刻切热点。

## 2. 登录与准备
- 访问 `github.com` 并登录。
- 所有操作在 **Chrome / Safari** 浏览器中进行，不需要安装任何桌面软件。

## 3. 创建新仓库
- 点击右上角 `+` → `New repository`（新建仓库）。
- 填写仓库名（如 `study-notes`）。
- ⚠️ **必须勾选** `Add a README file`（否则编辑器会报错）。
- 点击绿色 `Create repository`。

## 4. 编辑文件（两种方式）
| 方式 | 操作 | 适用场景 |
|------|------|----------|
| **网页直接编辑** | 点击文件名 → 铅笔图标 ✏️ → 改内容 → 滑到底部提交 | 改单个文件，网络不稳定时 |
| **vscode.dev 编辑器** | 地址栏把 `github.com` 换成 `vscode.dev` | 多文件操作、新建文件夹 |

## 5. 使用 vscode.dev 的关键步骤
- **打开**：`https://vscode.dev/github/用户名/仓库名`
- **授权**：首次使用会弹出“授权”页面，点击绿色按钮允许。
- **信任**：如果提示“受限模式”，点击“管理工作区信任”并选择“信任”。

## 6. 核心操作：提交并推送（最重要！）
> ⚠️ **记住这个口诀：改完 → 提交 → 推送**

1. 修改或新建文件后，点击左侧 **“源代码管理”** 图标（分支符号 ⑂）。
2. 在输入框写备注（如“更新了README”）。
3. 点击 **“提交并推送”**（蓝色按钮，一步到位）。
4. 去网页刷新仓库，确认文件已更新。

## 7. 新建文件夹 / 移动文件
- 在资源管理器里，在根目录或文件夹内点击 **“新建文件夹”**（📁+）。
- **移动文件**：在文件上右键/长按 → 选择“重命名”，在名字前加上文件夹名和斜杠（例如 `git/文件名.md`），确认后提交推送即可。

## 8. 常用 Markdown 语法
- 标题：`# 一级`、`## 二级`
- 无序列表：`- 项目`
- 待办清单：`- [ ]`（未完成）、`- [x]`（已完成）
- 代码块：``` 代码 ```

## 9. 遇到问题怎么办？
- **文件提交后网页没变化**：是否忘了点“提交并推送”？
- **编辑器打不开/报错**：仓库是不是空的？先在网页创建 `README.md`。
- **网络卡住**：立刻切手机热点，刷新页面重试。

## 10. 推荐工作流
1. 开热点上网。
2. 用 `vscode.dev` 编辑。
3. 改完点“提交并推送”。
4. 回网页看一眼，确认同步成功。

**祝你使用愉快！遇到问题随时回来查这份手册。**
